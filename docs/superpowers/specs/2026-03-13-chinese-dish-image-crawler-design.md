# 中餐菜品图片抓取工程设计

## 目标

构建一个本地批处理工程，用于为餐饮点餐系统准备中餐菜品图片素材。工程需要自动整理一份 50-100 个大众中餐馆常见菜品名称，按菜名从公开网络图片搜索结果中抓取图片，筛选并处理后输出到指定目录，同时生成一份 CSV 清单，标注图片 ID 和菜品名称。每种菜品需要保留 3 张最终图片。

## 范围

### 本期包含

- 内置一份 50-100 个大众中餐馆常见菜品名称
- 按菜品名称搜索公开网络图片
- 下载候选图片并做基础有效性过滤
- 将图片统一处理为 800x800 像素的输出图
- 按菜品分目录保存图片
- 生成一份 `manifest.csv`，记录图片 ID 和菜品名称等信息
- 对网络失败、图片损坏、数量不足等情况输出日志

### 本期不包含

- 手机端扫码点餐页面开发
- PC 端点餐软件开发
- 后台管理页面
- 数据库持久化
- 人工审核界面
- 高级图像识别或内容理解模型筛图

## 已确认约束

- 菜品范围：大众中餐馆常见菜
- 菜名来源：工程内置网络整理后的菜品清单
- 标注文件格式：CSV
- 输出图片规格：统一为 800x800
- 保存方式：按菜品分目录保存
- 每种菜品目标数量：3 张

## 运行形态

工程采用本地命令行批处理方式运行，不提供图形界面。默认执行流程如下：

1. 读取内置菜品清单
2. 对每个菜品搜索公开图片
3. 下载候选图片
4. 过滤无效或不合格图片
5. 裁剪并压缩为 800x800 成品图
6. 按菜品目录保存
7. 生成 `manifest.csv`
8. 输出统计和异常日志

## 核心流程设计

### 1. 菜品清单准备

- 维护一份 `data/dishes.txt`
- 文件中保存 50-100 个大众中餐馆常见菜名
- 第一版默认由工程内置，不依赖数据库或外部输入文件

### 2. 图片搜索

- 以菜名作为主关键词发起公开图片搜索
- 搜索词中附加例如“菜品图”“实拍”“中餐”等限定词，尽量减少抓到插画、门店广告、Logo、菜单截图
- 对每个菜品拉取多于 3 张的候选图，以便后续过滤

### 3. 候选下载

- 下载图片前先校验 URL 基本合法性
- 下载过程中记录来源 URL
- 支持请求超时与有限次重试
- 某一候选图失败时不影响当前菜品的后续处理

### 4. 图片过滤

基础过滤规则：

- 跳过无法下载的资源
- 跳过不是图片的响应内容
- 跳过损坏或无法解码的文件
- 跳过分辨率过低的图片
- 跳过尺寸明显不适合裁切的图片

第一版只做基础规则过滤，不引入复杂识别模型。若候选结果出现广告图、水印图或明显无关图，将依赖搜索关键词和数量补抓来提高通过率。

### 5. 图片处理

- 输出统一规格为 800x800
- 采用居中裁剪加缩放的处理方式
- 输出为统一常见格式，优先使用 JPEG
- 文件命名使用固定三位编号：`001.jpg`、`002.jpg`、`003.jpg`

### 6. 成品保存

- 输出目录为 `output/images/<菜品名>/`
- 每个菜品目录下最多保存 3 张最终图片
- 如果某个菜品不足 3 张，允许任务继续，最终在日志中列出缺失项

### 7. 清单导出

生成 `output/manifest.csv`，用于后续给点餐系统导入或人工核对。

建议字段如下：

- `image_id`：全局唯一图片编号
- `dish_name`：菜品名称
- `file_path`：输出图片相对路径
- `width`：输出宽度
- `height`：输出高度
- `source_url`：原始图片地址
- `status`：处理状态

示例：

```csv
image_id,dish_name,file_path,width,height,source_url,status
img_000001,鱼香肉丝,output/images/鱼香肉丝/001.jpg,800,800,https://example.com/a.jpg,ok
```

## 容错与恢复策略

- 网络请求失败时自动重试有限次数
- 单张图失败只记录并跳过，不中断整批任务
- 单个菜品最终不足 3 张时，在任务结束时汇总报告
- 已成功输出的图片默认不重复生成，便于后续增量补跑

## 质量标准

- 每个菜品尽量产出 3 张成品图
- 成品图必须能正常打开，像素固定为 800x800
- `manifest.csv` 中每条记录都应能对应到一个实际文件
- 图片路径和菜品目录结构清晰稳定，便于后续导入餐饮系统

## 风险与后续扩展

### 当前风险

- 公开搜索结果质量不稳定，部分菜品可能抓到无关图片
- 某些菜品图片来源较少，可能无法稳定拿到 3 张高质量图片
- 图片版权与来源合规需在业务使用前自行确认

### 后续扩展方向

- 增加指定站点抓取适配器
- 增加人工审核和替换机制
- 增加 SQLite 记录任务状态，实现更强的断点续跑
- 同时导出多种尺寸版本，适配不同终端

## 交付结果

完成后，工程应能输出：

- 一份内置大众中餐菜品名称清单
- 一批按菜品分类的 800x800 图片
- 一份 `manifest.csv` 文件
- 一份任务日志，列出成功、失败和不足 3 张的菜品
